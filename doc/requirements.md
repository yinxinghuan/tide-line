# Requirements

## 1. Overview

Wild Line 是一个 `social` 类型的移动端小游戏：刷到一片受人类活动威胁的栖息地（海滩/森林…），手指擦走垃圾陷阱、解救被困的野生动物——你救出谁谁就回到这片家园，做成可收藏邮票拼进全平台。剪纸手绘风，英文默认多语，零教程即点即玩。

## 2. Visual Design

- 整体布局：页面占用 900 x 720 px，主体验居中，HUD 与操作区覆盖在游戏层上方，移动端以单手操作为优先。
- 背景与配色：主要颜色使用 #1d2620、#2a1d16、#0000、#000、#ffffff、#f5a23c、#0b1820、#3f6238；高亮元素用于可点击目标、得分、结果或稀有状态。
- 字体：使用 'Inter', -apple-system, system-ui, sans-serif、'Inter', -apple-system, BlinkMacSystemFont, sans-serif、@sans、@mono，按钮与状态文字保持 12-24 px 的可读范围。
- 动画：常规按钮/卡片反馈控制在 120-240 ms；结果、命中、生成或翻牌反馈控制在 300-900 ms。
- 视觉元素：主对象保持在屏幕中心 40%-65% 视觉区域内；顶部/底部 HUD 保留至少 12 px 安全边距；可滚动墙或列表卡片使用固定间距，避免文本挤压。
- 美术素材清单：
- _creature_review_v2.png：位图图片，用于角色、场景、封面、反馈或品牌视觉。
- _creature_review.png：位图图片，用于角色、场景、封面、反馈或品牌视觉。
- _hedgehog_v3.png：位图图片，用于角色、场景、封面、反馈或品牌视觉。
- poster.png：位图图片，用于角色、场景、封面、反馈或品牌视觉。

## 3. Game Mechanics

- 初始化参数：
- `HORIZON`：0.42
- `MAX_VOICES`：12
- `MAX_LEN`：140
- `MAX_STORED`：30
- 更新循环：使用 requestAnimationFrame 驱动逐帧更新，目标刷新频率 60 FPS。 使用定时器推进倒计时、生成节奏或阶段切换。
- 核心机制：玩家完成主操作后更新分数、阶段、生成结果或收藏状态；反馈必须在 200 ms 内出现。
- 碰撞 / 命中：若存在运动目标，使用目标边界、距离或格子索引判断；命中后更新得分/连击，失误后扣除生命、时间或进入失败状态。
- 特殊机制：包含公共墙/画廊/归档浏览，玩家可以查看他人结果。 不依赖 AI 生成作为每局必需结果。
- 粒子 / 特效：命中、完成、生成、失败等关键事件使用上浮文字、闪光、缩放、抖动或淡出效果，单次特效 300-900 ms。

## 4. Controls

- Pointer：按下主操作区立即触发核心动作，单次 pointerdown 只计算 1 次。
- Click：用于按钮、卡片、结果项和可滚动列表里的选择确认。
- Keyboard：键盘事件用于桌面调试或方向/确认操作。
- Drag / Move：记录指针坐标变化，用于拖拽、瞄准、绘制或移动角色。

## 5. Win / Lose Conditions

- 达成目标后进入胜利/完成状态。
- 生命值/血量归零触发失败。
- 倒计时结束触发结算。
- 结算界面展示最终结果、历史最好或收藏结果，并提供再来一次、返回首页或继续浏览入口。

## 6. Sound Effects

- 主操作成功：合成短促提示音，正弦/三角波，约 440-880 Hz，80-160 ms。
- 失败或结束：低频下行提示，约 180-320 Hz，180-320 ms。
- 连击或奖励：上行音阶，约 660-1200 Hz，60-140 ms。
